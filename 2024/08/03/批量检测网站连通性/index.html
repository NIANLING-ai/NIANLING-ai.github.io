优化点详细解释：

  1. 数组管理 URL 列表：
    • 使用 Bash 数组存储 URL 列表，更易于管理和扩展。
  2. 增加参数支持：
    • 使用 getopts 命令解析命令行参数，允许用户自定义重试次数、超时时间和日志文件路径。
  3. 日志记录：
    • 错误日志记录到指定文件（默认是 url_check.log），方便后续检查。
  4. 并发处理：
    • 使用 & 将 URL 检查任务放入后台，并使用 wait 等待所有任务完成，提高效率。
  5. 改进输出格式：
    • 在输出中加入时间戳，以便更好地跟踪和分析。
  6. 帮助信息：
    • 添加了 -h 参数，可以显示帮助信息，方便用户查看脚本的使用方法。
