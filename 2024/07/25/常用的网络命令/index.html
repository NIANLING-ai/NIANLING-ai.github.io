  • 常用参数

  2. 执行远程主机上的命令
  3. 使用Ctrl+] 进入telnet命令模式
  4. 在命令模式下,输入quit断开连接并退出

网络连通性测试

  • 端口开放性检查
    如果能成功连接，说明目标IP和端口是开放的。如果返回”Connection refused”，表示网络可达但目标端口未开放。
  • 网络可达性测试
    如果返回”Connected to [IP]”，表示网络连接正常。如果返回”Connection timed out”，说明网络不通或被防火墙阻止。
  • 服务状态检查
    如果连接后立即返回”Connection closed by foreign host”，通常表示服务正在运行但拒绝了连接。

traceroute（tracert）

traceroute（在某些系统上(windows)也称为tracert）是一个网络诊断工具，用于确定数据包从一个源IP地址到一个目的IP地址的路径。它通过发送一系列的UDP数据包，每个数据包的TTL（Time To Live，生存时间）值逐渐增加，以便在路由路径上的每一跳（路由器）都返回一个ICMP时间超时消息（Time Exceeded）给发送者。这样，traceroute就可以列出经过的每一跳及其延迟。

使用方法

    • 每行显示一跳（路由器），按照它们从源到目标的顺序。
    • 每一行显示了路由器的IP地址（或者如果可能，显示其域名），以及数据包从源到达该路由器所经历的时间（延迟）。
  3. 选项：
    -n：以IP地址而非主机名显示跟踪。
    -q <查询次数>：指定每个跳跃的查询次数。
    -m <最大跳数>：设置最大跳数，以避免无限循环。
    -w <超时秒数>：设置超时时间，以等待每个回应。

nc命令

nc（Netcat）是一个功能强大的网络工具，常用于网络调试和数据传输。

基本概念

Netcat 是一个网络工具，可以用来：

  • 进行 TCP 或 UDP 连接
  • 监听端口
  • 发送和接收数据
  • 进行端口扫描
  • 作为简单的聊天工具

安装 Netcat

在大多数 Linux 发行版中，Netcat 通常预装。如果没有，可以通过包管理器安装：

  • Debian/Ubuntu: sudo apt-get install netcat -y
  • CentOS/RHEL: sudo yum install nc -y
  • macOS: brew install netcat -y

常见选项

-l: 监听模式
-p: 指定端口（在监听模式下使用）
-u: 使用 UDP 而不是 TCP
-v: 显示详细输出
-z: 扫描模式，不发送数据，只检查端口是否开放
-w: 设置超时时间（单位：秒）
